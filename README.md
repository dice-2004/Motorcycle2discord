# Motorcycle2Discord: ヘッドレスDiscord + RPC自動ステータス更新サーバー

本プロジェクトは、iPhoneのショートカットからのWebリクエストをトリガーにして、Dockerコンテナ内で公式Discord（Linux版）をオンデマンドで起動・停止し、Discord RPC経由であなたのステータス（「○○をプレイ中」）に「運転中」や「仕事中」などの任意のアクティビティを自動表示させるシステムです。

---

## 1. 特徴

* **完全に安全（規約準拠）**: 公式のLinux版Discordアプリと公式RPCの仕組みを使用するため、User Token（Self-bot）方式のようなアカウントBANリスクがありません。
* **低オーバーヘッド**: 運転中のみDiscordを起動し、停止中はプロセスを完全にキルするため、非運転時のサーバー負荷はほぼゼロ（メモリ消費10MB以下）です。
* **Webブラウザだけで設定可能**: `noVNC` を搭載しているため、特別なVNCアプリをインストールすることなく、お使いのWebブラウザ（ChromeやSafari等）だけで初回のログイン操作が行えます。
* **高い拡張性**: `config.json` を編集するだけで、Goのコードを変更することなく、新しいアクティビティ（仕事中、勉強中、睡眠中など）を簡単に追加・制御できます。

---

## 2. 準備するもの

### ① Discord Developer Portalでのアプリ作成
表示させたいアクティビティ名（例: `運転中`）を持つDiscordアプリを作成し、その「Application ID」を取得します。

1. [Discord Developer Portal](https://discord.com/developers/applications) にログインします。
2. **New Application** をクリックし、名前に「**運転中**」と入力して作成します。
3. **Application ID (Client ID)** をコピーしておきます。（これが設定ファイルに記述するIDになります。）
4. ※別のステータス（例: `仕事中`）も追加したい場合は、同様に「仕事中」という名前で別のアプリケーションを作成し、そのIDも控えておきます。

## 3. セットアップ手順

### 3.1 設定ファイルの作成 (`config/config.json`)
`config/config.json` を開き、ステップ2で取得したIDを設定します。

```json
{
  "activities": {
    "riding": {
      "name": "運転中",
      "details": "バイクを運転しています",
      "state": "インカム接続中 🏍️",
      "client_id": "取得したApplication IDをここに記述"
    }
  }
}
```

### 3.2 コンテナのビルドと起動
`docker-compose` を使用してコンテナを起動します。
*※競合を避けるため、デフォルトのホストポートは APIが `8089`、VNCが `6080` になっています。*

```bash
docker-compose up -d --build
```

### 3.3 初回のみ：WebブラウザによるDiscordへのログイン
Discordはログインが完了するまで通信ソケットを作らない仕様があるため、自動キル（タイムアウト）を防ぐために**一時的に直接Discordを起動して**ログイン操作を行います。

1. **コンテナ内でDiscordを直接起動する**:
   Dockerを起動しているLinuxターミナルで以下のコマンドを実行します。
   ```bash
   docker exec -it motorcycle2discord-rpc discord --multi-instance --no-sandbox --disable-gpu --disable-dev-shm-usage
   ```
   *※このコマンドで起動したDiscordはタイムアウトで終了しません。*

2. **WebブラウザでVNC画面を開く**:
   PC等のブラウザから `http://[ProxmoxのIPアドレス]:6080/vnc.html` を開きます。
   画面の「**Connect**」ボタンをクリックすると、Discordのログイン画面（QRコード）が表示されます。

3. **ログインを完了させる**:
   スマホのDiscordアプリで画面のQRコードをスキャンし、ログインを承認します。
   **【重要】ブラウザ上の画面が、ご自身のDiscordのチャット画面（フレンド一覧など）に完全に切り替わるまで待ってください。** 切り替わったら、セッションデータが確実に保存されるよう、そのまま15秒ほど待ちます。

4. **起動プロセスの終了**:
   ログインが完了したら、手順1でコマンドを実行したLinuxターミナルで **`Ctrl + C`** を押してコマンドを終了します（ブラウザ上の画面からDiscordが消えます）。

## 4. 使い方（iPhoneショートカットの設定）

Bluetoothインカム（または他のトリガー）の接続・切断に応じて、iPhoneの「ショートカット」アプリでオートメーションを設定します。

### ① 運転開始時（インカム接続時）のオートメーション
1. iPhoneで「ショートカット」アプリを開き、**オートメーション** タブを選択します。
2. **新規オートメーション** を作成し、トリガーに **Bluetooth ➔ [バイクのインカム名] ➔ に接続したとき** を選択し、「すぐに実行」にチェックを入れます。
3. アクションに **「URLの内容を取得」** を追加し、以下のように設定します。
   * **URL**: `http://[ProxmoxのIPアドレス]:8089/activities/start`
   * **方法**: `POST`
   * **要求本文**: `JSON`
   * **新規フィールドを追加** (「テキスト」を選択):
     * キー: `type`
     * 値: `riding`

### ② 運転終了時（インカム切断時）のオートメーション
1. 同様に、**新規オートメーション** を作成し、トリガーに **Bluetooth ➔ [バイクのインカム名] ➔ から切断したとき** を選択し、「すぐに実行」にします。
2. アクションに **「URLの内容を取得」** を追加し、以下のように設定します。
   * **URL**: `http://[ProxmoxのIPアドレス]:8089/activities/stop`
   * **方法**: `POST`
   * **要求本文**: `なし`（または空のJSON）

## 5. セキュリティと軽量化のベストプラクティス

初回ログインが完了した後は、安全のためVNCサーバー機能（ブラウザでの画面閲覧機能）を完全に停止させることを強く推奨します。

1. `docker-compose.yml` を開き、環境変数を変更します。
   ```yaml
   environment:
     - ENABLE_VNC=false  # true から false に変更
   ```
2. コンテナを再起動します。
   ```bash
   docker-compose up -d
   ```
これにより、コンテナ内のVNCサーバーや中継用プログラム（websockify）のプロセス自体が起動しなくなるため、**通常運用時のセキュリティが向上し、サーバーのオーバーヘッドも完全に「ゼロ」**になります。

## 6. 新しいアクティビティの追加方法

将来的に「仕事中」など別のアクティビティを増やしたい場合の手順です。

1. Developer Portalで「仕事中」というアプリを新規作成し、IDを取得します。
2. `config/config.json` に項目を追記します。
   ```json
   "working": {
     "name": "仕事中",
     "details": "コーディングに集中しています",
     "state": "連絡は遅れます 💻",
     "client_id": "取得した仕事中用のアプリアプリケーションID"
   }
   ```
3. iPhoneのショートカットで、本文の `type` の値を `working` に変更したリクエストを送るだけで、自動的に「仕事中」のステータスに切り替わります（Goプログラムの再ビルドは不要です）。
