## 1. 履歴

### 2026-07-01: 初期設計と要件定義
* **要件の整理**:
  * バイクのインカム接続時にiPhoneのショートカットからWebリクエストを送り、Discordのステータスに「運転中」と表示する。
  * Discord Bot方式（全サーバーにBot導入が必要）ではなく、ユーザー本人のアカウントでステータスを表示したい。
  * アカウントの安全性を考慮し、かつPCを常時起動しないために、「Docker内の仮想ディスプレイ(Xvfb)で公式Discordアプリを動かし、公式RPCを使ってステータスを更新する」方式（規約違反にならない安全な方式）を採用。
  * サーバー（Proxmox）のオーバーヘッドを削減するため、**「運転中のみDiscordを起動し、停止時はDiscordを終了する」**動的起動方式を採用。これにより非稼働時のメモリ消費を10MB以下に抑える。
  * 今後のアクティビティ増設に対応できるよう、APIを汎用化し、設定ファイルから動的にアクティビティ定義を読み込める拡張性の高い設計にする。
* **技術スタックの選定**:
  * **言語**: Go (軽量、低オーバーヘッド、シングルバイナリ)
  * **ベースOS**: `debian:bookworm-slim` (公式Discordアプリが確実に動作し、かつ最軽量)
* **ドキュメントの作成**:
  * `AGENTS.md` (コーディングルール) の作成完了。

### 2026-07-01: 設計書の作成とコード実装 (完了)
* **ドキュメントの整備**:
  * `AGENTS.md` (ルール), `work_log.md` (作業ログ), `spec.md` (仕様書), `README.md` (手順書) の作成。
* **Go APIサーバーの実装**:
  * `config.go`: `config.json` から動的にアクティビティ定義をパースする構造を実装。
  * `process.go`: Xvfbが常時稼働する前提で、Discordプロセスの起動・キル・状態監視をGo側から制御する設計にリファクタリング。
  * `rpc.go`: `rich-go/client` を使用して、Discord RPCに動的にClient IDを設定しステータス（活動名や開始時刻）を反映するロジックを実装。
  * `main.go`: HTTPルーターを立ち上げ、非同期でDiscordの起動・ステータス設定を処理する非同期ワークフローを実装（iPhoneショートカットのタイムアウト回避）。
* **Docker環境の構築**:
  * `Dockerfile`: `debian:bookworm-slim` をベースとし、ChromiumベースのDiscordに必要なGUI共有ライブラリ等を厳密にインストール。マルチステージビルドによる軽量化（Goコンパイル結果のみを配置）。
  * `entrypoint.sh`: 起動時に最新のDiscord `.deb` をダウンロードしインストールすることで強制アップデートを回避。XvfbとVNCサーバーを常時バックグラウンド起動。
  * `docker-compose.yml`: データ永続化ボリューム（ログインセッション用）とポートフォワード（8080/5900）の設定。
* **ローカル検証**:
  * `go build -o server` を実行し、Goプログラムがエラーなく完全にビルドされることを確認。
* **フォルダ構成の整理とルール更新**:
  * メンテナンス性の向上のため、ファイルを適切にフォルダ分け（`src/`, `config/`, `docs/`, `docker/`）して再配置。
  * フォルダ変更に伴い、`docker-compose.yml` および `docker/Dockerfile` 内のソースコード・設定ファイルパスを修正し、ビルドテストに成功。
  * `AGENTS.md` に「ユーザーコミュニケーション（確認・質問の徹底）」のルールを追加。
* **拡張性を考慮したインターフェース設計へのリファクタリング**:
  * 将来的に他のAIエージェントがDiscordを介して代理返答する機能や、別の接続手段（Direct API等）を導入することを見据え、`DiscordController` インターフェースを定義した `src/service.go` を導入。
  * `main.go` のHTTPルーティング側から具体的なプロセス・RPC操作の依存性を排除し、インターフェースを介した疎結合な設計にリファクタリング。
  * `docs/spec.md` に拡張設計および将来のチャット送受信機能の追加シナリオについて追記。
* **Dockerfileのビルドエラー修正 (Goバージョン不一致の修正)**:
  * ローカルのGoバージョン (`go.mod` に定義された `go 1.24.7`) と `Dockerfile` 内のビルドイメージ (`golang:1.22-bookworm`) の不一致により、`go mod download` が失敗していたエラーを修正。
  * `Dockerfile` のビルドステージベースイメージを `golang:1.24-bookworm` にアップグレードし、Goバージョン要件を満たすように変更。
* **Discordインストール時の依存関係エラーの修正**:
  * `entrypoint.sh` 内でのDiscordパッケージインストール時、`libatomic1`, `libgbm1`, `libnotify4` の不足により `dpkg` エラーとなり、`apt-get install -f` でDiscord自体が削除されてしまっていたバグを修正。
  * `Dockerfile` にこれらのライブラリを明記してビルド時にあらかじめインストールされるようにし、`entrypoint.sh` でも事前に `apt-get update` を行うことで、確実に依存関係が解消・インストールされるように修正。
* **DiscordのGPU関連フラグ追加 (無限ローディング対策)**:
  * 仮想ディスプレイ上でのGPU初期化エラーや共有メモリ不足による、Discordの「starting (launching)」無限ロード・フリーズ問題を修正。
  * `process.go` のDiscord起動引数に `--disable-gpu` および `--disable-dev-shm-usage` フラグを追加。
* **起動タイムアウト時間の延長 (30秒から90秒へ)**:
  * 初回起動時やネットワーク環境による起動遅延の際、Discordの自動ログイン（FAST CONNECT）が完了する前に30秒が経過し、Goサーバーがエラー判定を下してDiscordをキルしてしまう問題に対応。
  * `service.go` 内のソケット生成待ち時間を90秒に延長し、正常起動を安全に待てるように調整。
